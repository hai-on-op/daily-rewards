# Daily Rewards Dashboard — Data Specification v2

## Data Source

A single JSON file generated by `yarn daily-report`, saved to `reports/daily-reward-report-{timestamp}.json`. Regenerated daily. Typical size: ~3MB for 30 days / 49 users.

---

## JSON Schema

### Root Object

```typescript
{
  generatedAt: string;         // ISO 8601, e.g. "2026-03-25T14:56:00.640Z"
  periodDays: number;          // Requested period, e.g. 30
  totalDaysWithData: number;   // Actual days with data, e.g. 30
  globalAverages: GlobalAverages;
  dailyReports: DayReport[];   // One per day, chronological (oldest first)
  users: AggregatedUser[];     // One per user, sorted by total earned desc
}
```

### GlobalAverages

```typescript
{
  avgDailyRewardByToken: Record<string, number>;
  // e.g. { "HAI": 79.13, "KITE": 50.03 }

  avgBoostedPositions: number;
  // Average count of users with any boost > 1.0x per day

  avgDailyStrategyTotals: {
    strategy: string;    // "haiVELO" | "haiAERO" | "lpStaking" | "minter"
    token: string;       // "HAI" | "KITE"
    avgDailyTotal: number;
  }[];
}
```

**Note:** `lpStaking` and `minter` may appear multiple times (multiple pools/collateral types). Sum duplicates by strategy+token for display.

### DayReport

```typescript
{
  dayTimestamp: number;        // Unix seconds (midnight UTC)
  date: string;                // "YYYY-MM-DD"
  strategyTotals: {
    strategy: string;
    token: string;
    totalReward: number;
  }[];
  totalRewardByToken: Record<string, number>;
  totalBoostedPositions: number;
  users: Record<string, DayUserEntry>;  // keyed by lowercase 0x address
}
```

**Note:** `strategyTotals` can have duplicate strategy+token combos. Always sum them.

### DayUserEntry

```typescript
{
  dailyEarned: Record<string, number>;
  // Token totals. e.g. { "HAI": 47.81, "KITE": 4.68 }

  dailyStrategyEarned: Record<string, Record<string, number>>;
  // strategy → token → earned
  // e.g. { "haiVELO": { "HAI": 46.47 }, "minter": { "KITE": 30.48 } }

  dailyStrategyShare: Record<string, Record<string, number>>;
  // strategy → token → share (0–1 fraction)

  strategyPositions: Record<string, Record<string, StrategyPositionData>>;
  // strategy → token → position data (see below)

  kiteStaked: number;
  kiteShare: number;           // Fraction 0–1
  boosts: Record<string, number>;
  // Boost key → multiplier (1.0–2.0)
  // Keys: "minter", "haivelo", "haiaero", "lpStaking_HAI_BOLD_CURVE",
  //        "lpStaking_HAI_VELO_VELO", "lp"

  hasBoostedPosition: boolean;
}
```

### StrategyPositionData

The core data structure for understanding reward distribution per strategy per day.

```typescript
{
  avgWeight: number;
  // User's effective boosted weight for this day.
  // Derived from earned: avgWeight = userEarned (in token units).
  // avgWeight / avgTotalWeight = userEarned / strategyPoolTotal (exact match).

  avgUnboostedWeight: number;
  // User's effective weight WITHOUT boost = avgWeight / endOfDayBoost.
  // avgUnboostedWeight / avgTotalUnboostedWeight gives the user's share
  // of the pool IF nobody had any boost.

  avgTotalWeight: number;
  // Total effective boosted weight across ALL users in this strategy.
  // Equal to the strategy's pool total for this day (in token units).

  avgTotalUnboostedWeight: number;
  // Sum of (earned / boost) across ALL users in this strategy.
  // Represents what the total pool weight would be without boosts.

  endOfDayWeight: number;
  // Point-in-time position weight at end of day (debt, collateral, LP staked).
  // For multi-collateral strategies (minter), this is summed across types.
  // Informational — does NOT directly explain the daily earned amount
  // because rewards are time-weighted over the entire historical period.

  endOfDayBoost: number;
  // Boost multiplier at end of day (1.0–2.0).
}
```

**Key relationships:**
```
avgWeight / avgTotalWeight = earned / poolTotal           (boosted share, EXACT)
avgUnboostedWeight / avgTotalUnboostedWeight              (unboosted share)
boostedShare > unboostedShare when user's boost > average  (boost helps)
boostedShare < unboostedShare when user's boost < average  (boost hurts relatively)
```

### AggregatedUser

30-day averages per user. Unchanged from v1.

```typescript
{
  address: string;
  avgDailyEarnedByToken: Record<string, number>;
  avgDailyStrategyEarned: Record<string, Record<string, number>>;
  avgDailyStrategyShare: Record<string, Record<string, number>>;
  avgKiteStaked: number;
  avgKiteShare: number;
  avgBoosts: Record<string, number>;
  daysActive: number;          // 1–30, averages divide by this
}
```

## Derived Calculations (Client-Side)

### User's Overall Token Share
```
share = user.dailyEarned[token] / dayReport.totalRewardByToken[token]
```

### Boosted Share (per strategy)
```
boostedShare = pos.avgWeight / pos.avgTotalWeight
```
This equals `earned / poolTotal` exactly.

### Unboosted Share (per strategy)
```
unboostedShare = pos.avgUnboostedWeight / pos.avgTotalUnboostedWeight
```
Shows what the user's share would be without any boost effects.

### Max Boost Potential
What the user would earn at 2.0x boost, correctly accounting for the increased total pool:
```
s = pos.avgWeight / pos.avgTotalWeight        // current boosted share
b = pos.endOfDayBoost                          // current boost

maxShare = (2 * s) / (b * (1 - s) + 2 * s)
maxEarned = strategyPoolTotal * maxShare
additionalEarned = maxEarned - currentEarned
pctIncrease = additionalEarned / currentEarned
```
Only applicable when `endOfDayBoost < 2.0`. Show "Already at max boost" when = 2.0.

### Average Boost (for hero stats)
```
boostValues = Object.values(user.avgBoosts).filter(b => b > 1)
avgBoost = boostValues.length > 0 ? sum(boostValues) / boostValues.length : 1
```

## Dashboard Sections

### 1. Address Search
- Text input for `0x` + 40 hex chars
- Case-insensitive: lowercase input, match `users[].address`
- Show "not found" state if no match
- No wallet connection required

**Source:** `globalAverages` + `users.length` + `totalDaysWithData`

| Card | Source |
|------|--------|
| Avg Daily HAI | `globalAverages.avgDailyRewardByToken.HAI` |
| Avg Daily KITE | `globalAverages.avgDailyRewardByToken.KITE` |
| Rewarded Users | `users.length` |
| Avg Boosted Positions | `globalAverages.avgBoostedPositions` |
| Report Period | `totalDaysWithData` + " days" |

Strategy breakdown table from `avgDailyStrategyTotals` (aggregate by strategy+token).

### 3. User Hero Stats
**Source:** `AggregatedUser`

| Card | Source |
|------|--------|
| Avg Daily HAI | `avgDailyEarnedByToken.HAI` |
| Avg Daily KITE | `avgDailyEarnedByToken.KITE` |
| Days Active | `daysActive` / 30 |
| Avg Boost | Derived (see formula) |
| KITE Staked | `avgKiteStaked` |
| KITE Share | `avgKiteShare` as % |

### 4. Earnings Chart
**Source:** `dailyReports[].users[address].dailyEarned`

Two-line chart: HAI (cyan `#22d3ee`) and KITE (green `#10b981`) per day.

### 5. Strategy Breakdown (30-day avg)
**Source:** `AggregatedUser.avgDailyStrategyEarned` + `avgDailyStrategyShare`

| Strategy | Token | Avg Daily Earned | Avg Pool Share |
|----------|-------|-----------------|----------------|

### 6. Boost Section (30-day avg)
**Source:** `AggregatedUser.avgBoosts` + `avgKiteStaked` + `avgKiteShare`

Visual horizontal bars (0–100% where 100% = 2.0x):
- Width: `(boost - 1) * 100%`
- Color: >=1.8 green, >=1.3 amber, else dim

### 7. Daily Detail Cards
One card per day, newest first (with sort toggle).

#### 7a. Narrative Summary
Human-readable story with values right-aligned:

```
"You earned a total of"                                    → {sum of tokens}
"Out of {pool} {TOKEN} distributed, you captured"          → {share%}
"In {strategy}, your avg weight of {avgWeight}             → {earned} {TOKEN}
  ({share%} of pool, {boost}x boost) earned you"
"If you had max boost in {strategy}, you would have        → +{extra} {TOKEN}
  earned an extra"
"You had {kiteStaked} KITE staked, giving you"             → {kiteShare%}
```

For max boost line, only show when `endOfDayBoost < 2.0` and extra > 0.0001.

#### 7b. Token Summary Table

| Token | Pool Total | You Earned | Your Share |
|-------|-----------|------------|------------|

**Source:** `dayReport.totalRewardByToken` vs `userEntry.dailyEarned`
Share = earned / pool.

#### 7c. Strategy Blocks
One block per active strategy. Each contains:

**Header row:**
```
[Strategy Name] [TOKEN] Pool: {poolTotal} You: {earned} [{share%}] [{boost}x]
```
- Strategy name: display name (see mapping below)
- Pool total: sum of `strategyTotals` for this strategy+token
- Share: `earned / poolTotal`
- Boost badge: `endOfDayBoost`, colored green >=1.5, amber >=1.2

**Weights table** (when `strategyPositions` exists and earned > 0):

|           | Total Weight | Your Avg Weight | Your Share |
|-----------|-------------|-----------------|------------|
| Unboosted | `avgTotalUnboostedWeight` | `avgUnboostedWeight` | `avgUnboostedWeight / avgTotalUnboostedWeight` |
| Boosted   | `avgTotalWeight` | `avgWeight` | `avgWeight / avgTotalWeight` |

Boosted share should match earned/pool exactly. Unboosted share shows the user's position strength without boost effects — the difference between rows reveals how much boost is helping.

**End-of-day position** (when `endOfDayWeight > 0`):
```
End-of-day position: {endOfDayWeight} x {endOfDayBoost}x = {endOfDayWeight * endOfDayBoost} boosted
```
Informational only — shows current position size.

**Max boost line** (when `endOfDayBoost < 2.0`):
```
At max boost (2.0x): +{extra} {TOKEN} (+{pct}% more) = {maxTotal} total
```

**Already at max** (when `endOfDayBoost >= 2.0`):
```
Already at max boost
```

#### 7d. Footer
- Boost tags: colored pills per boost key
- KITE Staked + KITE Share

## Value Ranges & Formatting

| Field | Range | Format |
|-------|-------|--------|
| Token earned/day | 0 – 50 | >=1000: 1dp, >=1: 3dp, >=0.001: 5dp, else scientific |
| Strategy share | 0 – 0.53 | As %: >=1% → 1dp, >=0.01% → 2dp |
| Boost | 1.0 – 2.0 | 2dp + "x". Color: >=1.5 green, >=1.2 amber |
| KITE staked | 0 – 25,236 | Same as token amount |
| KITE share | 0 – 0.109 | Same as strategy share |
| avgWeight / avgTotalWeight | Earned amounts | Same as token amount |
| endOfDayWeight | Varies (debt, collateral, LP) | Same as token amount |
| Days active | 1 – 30 | Integer |

## Important Notes

### Multi-Sub-Entry Strategies
Minter has 6 collateral types and lpStaking has 2 pools. Each produces separate entries in `strategyTotals`. The `strategyPositions` data is correctly aggregated:
- `avgWeight` / `avgTotalWeight` = exact `earned / pool` ratio
- `avgTotalUnboostedWeight` = sum of `(earned / boost)` across ALL users
- `endOfDayWeight` = summed across sub-entries

### HAI Gaps in Last ~7 Days
haiVELO and haiAERO use weekly transfer epochs. The last epoch ends ~7 days before the report date. Days after that only have KITE (lpStaking/minter) rewards. This is by design, not a bug.

### Address Lookup
1. Validate: `/^0x[a-fA-F0-9]{40}$/`
2. Lowercase
3. Find in `users[]` by `address` → `AggregatedUser`
4. Filter `dailyReports[]` to days where `users[address]` exists
5. If not found: show warning

### endOfDayWeight vs avgWeight
- `avgWeight` = effective weight for earned calculation (= earned amount). Use for shares and max boost.
- `endOfDayWeight` = actual position size at midnight (debt, collateral, LP staked). Informational only — does NOT directly explain earned because rewards are time-weighted across historical events.
